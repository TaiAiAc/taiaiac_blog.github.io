MVVM

  • 架构方式、开发思想
  • M：Model数据模型
  • V：View视图界面（HTML）
  • VM：ViewModel视图数据模型
    • 驱动视图改变data数据

M：Model

  • 服务端处理数据库交互
    • mongoose
    • 设计数据模型
    • models
    • find
    • new User().save()
  • 客户端要处理服务器的接口交互
    • ajax(‘地址’)
    • GET http://127.0.0.1:3000/users
    • POST http://127.0.0.1:3000/users/1
    • PATCH http://127.0.0.1:3000/users/1
    • DELETE http://127.0.0.1:3000/users/1
    • 没有表意性
      • User.find()

V：View

VM：ViewModel

优点

  • 低耦合。视图（View）可以独立于Model变化和修改，一个VIewModel可以绑定到不同的“View”上，当View变化的时候Model可以不变，当Model变化的时候View也可以不变
  • 可重用性。你可以把一些视图逻辑放到一个VIewModel里面，让很多View重用这段视图逻辑。
  • 独立开发。开发人员可以专注于业务逻辑和数据的开发（ViewModel），设计人员可以专注于页面设计，使用Expression Blend可以很容易设计界面并生成xml代码
  • 可测试。界面素来是比较难于测试的，而现在测试可以针对ViewModel来写。

VM原理

  • 事件观察者
  • 事件发布 / 订阅模式
  • Object.defineProperty高级属性定义方式
  • 所有关心message改变的 DOM 都订阅了 message 事件
  • 当 message 改变的时候，就发布 message 事件
