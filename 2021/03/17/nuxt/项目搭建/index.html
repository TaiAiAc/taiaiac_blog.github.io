项目搭建

Author: Z.A.M. Date: March 17, 2021  13:51:50 Category: nuxtjs

1.命令行创建项目

  • npx create-nuxt-app <项目名>
  • yarn create nuxt-app <项目名>

2.选项配置

项目配置：

  • 语言: JavaScript

  • 包管理 : yarn

  • UI框架：Element

  • 使用集成的 : Axios - Promise based HTTP client

  • 代码检测 : Prettier

  • 测试框架：None

  • Nuxt模式：Universal (服务端渲染)

  • 部署目标 : 服务器 or 静态托管

  • 开发工具 : jsconfig.json(vs code)

  • 版本控制 : git

3.项目目录

重要目录

  • .nuxt
    • 打包后的文件,如查看router生成后的文件
  • assets
    • 全局文件目录css,js,img…..
  • components
    • 全局组件
  • layouts
    • default.vue为全局默认布局
    • 其他布局,和组件调用方式一致
  • middleware
    • 中间件目录
    • auth.js一般作为全局路由使用
  • pages
    • 项目目录即是路由视图
    • 约定式路由,自动生成路由视图
  • plugins
    • 插件目录
    • 全局资源配置
  • static
    • 静态资源目录
    • 无需打包的文件存放地
  • store
    • vuex状态树
    • 保存状态,记录操作….
  • utils
    • 工具
    • 一些封装的拿来即用的方法函数
  • nuxt.config.js
    • 项目配置目录
    • 很重要
  • dist
    • yarn generate命令生成静态部署包
    • 客户端渲染的包

4.环境搭建

安装项目必备的插件

  • yarn add @nuxtjs/axios @nuxtjs/proxy cookie-universal-nuxt
    • axios,配置代理,cookie工具
  • yarn add node-sass sass-loader scss-loader
    • css预处理器
  • @nuxtjs/style-resources
    • sass,less变量全局使用插件,无需引入

nuxt.config.js配置

  • server
    • port 端口
    • host 地址
  • head
    • 全局head标签内容
  • css
    • 全局css的引入
  • router
    • middleware 全局中间件
    • 路由配置
  • plugins
    • 全局插件注入,注意前后顺序
    • 引入plugins目录的插件
  • loading
    • 定义系统默认loading效果,或者指定loading组件
    • loading: { color: '#007fff', height: '50px' }
    • loading: '~/components/loading.vue'
  • modules
    • 模块
    • modules: ['@nuxtjs/axios', 'cookie-universal-nuxt', '@nuxtjs/style-resources']
  • axios
    • axios相关配置
  • proxy
    • 配置代理
  • components
    • components: true
    • 开启全局组件,无需引用直接使用
  • styleResources
    • 全局样式变量
    • scss: ['~/assets/scss/variable.scss']

5.路由

  • layouts/default.vue中<nuxt />下的为一级路由视图

  • 一级路由即pages目录下的.vue文件

  • 二级路由视图标签为<NuxtChild />

  • 约定式路由中实现重定向

    • index.vue为默认展示页,也就是重定向
    • 重定向需要与父路由名相同的文件目录下的index.vue
    • 也就是pages/home.vue中的<NuxtChild />默认显示的页面为pages/home/index.vue
    • 同一目录下跳转为同级路由跳转,会按照父路由展示点展示

  • 动态路由

    • 动态路由的页面必须在文件名前加_
    • 如给_id.vue动态传参
    • 字符串方式
      • home/see/1?a=1&b=2 home/see/2?a=11&b=22
    • 对象方式(params对等文件名,传递的是id)
      • {name:'home-id',params:3,query:{a:111,b:222}}

  • 路由传值

    • 使用asyncData接受到的值可以直接使用
