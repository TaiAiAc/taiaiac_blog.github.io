GitHub上部署hexo博客

今天gitee上部署的博客被莫名其妙封禁了,也没办法重新部署,只好在github上重新部署了一下

1.创建仓库

  • 右上角创建新仓库
  • 仓库名+.github.io
  • settings
  • Pages
  • 启用Pages服务
  • 添加基础配置

2.更改配置文件 _config.yml

  • 配置项目地址url
    • 项目地址为仓库地址
    • 如 : https://github.com/TaiAiAc/taiaiac_blog.github.io
  • 配置项目根目录root
    • 根目录为仓库名
    • 如 : /taiaiac_blog.github.io
  • 配置仓库地址 repo
    • 仓库地址
    • 如 : https://github.com/TaiAiAc/taiaiac_blog.github.io

3.配置自定义域名

前置:需要拥有域名

一级域名配置

  • 需要dns解析
  • 解析后添加记录
  • 添加远程仓库地址
  • 添加一级域名解析

需要终端ping+仓库名,得到的ip填入@的记录值中

主机记录 记录类型 记录值
@ A 如: 0.0.0.0
www CNAME 如: taiaiac_blog.github.io.

二级域名配置

多一步骤 , 增加一条解析二级域名的操作

主机记录 记录类型 记录值
@ A 如: 0.0.0.0
www CNAME 如: taiaiac_blog.github.io.
如: zhang CNAME 如: taiaiac_blog.github.io.

4.添加远程地址

自定义域名后需要再次修改配置

  • url 修改为当前域名地址
  • root 修改为 /
  • 远程仓库地址不变

5.添加CNAME文件

自定义域名通过CNAME文件中填写的地址为博客地址,hexo每次更新都会覆盖仓库内容,需要每次手动添加CNAME文件

文件内容为

如 : www.baidu.com

填写:

# hexo
vscode插件
霓虹灯效果
